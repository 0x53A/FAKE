What's new in FAKE 5?

FAKE 5 was build on several design principles ...

Modularization

Only pay for what you need

Extendibility

Integrate your own modules

Quickstart

Get started as fast as possible

All these principlies lead to a new way of using FAKE 5. This will open a lot of opportunites for FAKE to go where it never has gone before.

Migration

There is a clear migration path to FAKE 5

No, your current build scripts will not break, but the development of new feature for FAKE 4 and the old way of using FAKE are no longer supported.

Start Migrating

Bootstrapping

FAKE 5 now supports a lot more bootstrapping options

Every project has different needs. Therefore FAKE 5 supports several ways to bootstrap the build process. From an assumed installation - for example via chocolatey - to automatically updated bootstrapping scripts.
